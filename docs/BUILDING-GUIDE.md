# ビルガイド

GitCity Henkakuのビル仕様と、PRでカスタマイズする方法。

## ビルの仕組み

各受講生に1棟のビルが割り当てられる。ビルの形状・色は受講生のアクティビティを反映する。

### ビル属性の意味

| 属性 | 意味 | データソース |
|------|------|------------|
| **高さ** | トークン保有量 | Optimismチェーン |
| **幅** | GitHubのcommit数 | GitHub API |
| **奥行き** | GitHubのPR数 | GitHub API |
| **色** | トークン量のグラデーション | Optimismチェーン |
| **窓の明かり** | 直近7日間のGitHub活動 | GitHub API |

### 配置ルール

ビルはトークン総数の多い順に中心から外側へスパイラル状に配置される。

```
         ┌───┐ ┌───┐
    ┌───┐│   │ │   │┌───┐
    │   ││ 2 │ │ 3 ││   │
    │ 8 │└───┘ └───┘│ 4 │
    └───┘ ┌───┐     └───┘
          │ 1 │           ← 1が中心（トークン最多）
     ┌───┐└───┘┌───┐
     │ 7 │     │ 5 │
     └───┘┌───┐└───┘
          │ 6 │
          └───┘
```

## ビルのカスタマイズ（PRで追加）

ビルの描画ロジックを拡張して、新しいスタイルや装飾を追加できる。

### 変更可能な範囲

**追加できるもの:**
- ビルの装飾（屋上、看板、アンテナ等）
- 色のテーマ（ダークモード対応等）
- アニメーション効果
- 条件付きスタイル（例: トークンが一定以上で特別な装飾）

**変更しないもの:**
- ビル属性の計算ロジック（`city-math.ts`）— 公平性のため
- 配置アルゴリズム（`CityLayout.ts`）— 一貫性のため
- データの型定義（`types/index.ts`）— 互換性のため

### PRの手順

1. [CONTRIBUTING.md](../CONTRIBUTING.md) の手順でローカル環境をセットアップ
2. `src/components/city/` 内のファイルを変更
3. `npm run dev` でサンプルデータを使って動作確認
4. PRを出す（スクリーンショット添付推奨）

### 関連ファイル

| ファイル | 役割 |
|---------|------|
| `src/components/city/CityCanvas.tsx` | City全体の描画 |
| `src/components/city/Building.ts` | ビルの描画ロジック |
| `src/components/city/CityLayout.ts` | 配置アルゴリズム |
| `src/components/city/WaterwayRenderer.ts` | 水路の描画 |
| `src/components/city/CityTooltip.tsx` | ホバー時の情報表示 |
| `src/lib/city-math.ts` | データ→ビル属性の変換 |

## 水路

ビル間の水路は貢献トークンの送受信を表す。

- **太さ** = 累計流通量
- **色の鮮やかさ** = 流通の新しさ
- **方向** = トークンの流れ

水路が多いビル = コミュニティの中心。水路がないビル = まだ誰ともトークンを送り合っていない。

## ローカル開発のヒント

- `data/students.sample.json` にサンプルデータがある（10人分の架空データ）
- 実際のOptimismデータは不要 — サンプルだけで描画を確認できる
- ブラウザの開発者ツールでCanvasの描画を確認できる
